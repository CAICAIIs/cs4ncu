### **`CS for NCU` 项目全景报告**

#### 一、项目概览 (Executive Summary)

`CS for NCU` 是一个由南昌大学超算俱乐部 (NCUSCC) 发起、由广大社区贡献者共同维护的开源知识库项目。其核心定位是**一份有温度、有深度的南昌大学计算机学习指北**。它并非简单的技术文档罗列，而是一个旨在**传承经验、扫清障碍、指明方向**的“活”的生态系统，致力于为南昌大学乃至更广泛的计算机初学者铺就一条清晰、温暖的成长之路。

#### 二、核心哲学与使命 (The "Why")

项目的灵魂在于其独特的内容组织哲学，这是经过反复迭代最终确立的，包含三大基石：

1.  **“学习者路径”优先原则**：我们摒弃了传统的按技术领域划分的目录，严格遵循一个新手的认知和成长路径来组织内容，确保学习流程的**线性与连贯性**。
2.  **“主线与附录”分离原则**：这是项目结构设计的核心。
    *   **主线**只保留对所有人都至关重要的**普适性知识**。
    *   **附录**则收纳所有**面向特定人群、规则复杂、时效性强**的“攻略”，如保研、考研等。
    This keeps the core content clean and stable.
3.  **“团队专长”高亮原则**：项目明确划分出“团队特色研究”模块，用于展示核心维护团队在特定工具（如 `VS Code`）上的深度研究，以此提供独特的、经过实践检验的价值。

#### 三、内容架构 (The "What")

基于上述哲学，项目的内容被清晰地划分为三大板块，这一结构直接体现在网站的顶层导航上：

1.  **核心成长路径 (Main Quest)**
    *   **定位**：新手的必经之路，构成项目的主干。
    *   **内容**：`章一：告别电脑小白` -> `章二：规划你的未来` -> `章三：在 NCU 高效学习` -> `章四：硬核技能入门`。

2.  **团队特色研究 (Featured Content)**
    *   **定位**：核心团队的“私房菜”，提供深度和独特见解。
    *   **内容**：当前聚焦于 `专题：终极编辑器 VS Code`。

3.  **昌大专属附录 (Appendices)**
    *   **定位**：可随时查阅的、针对性极强的“参考手册”。
    *   **内容**：`保研细则`、`考研经验`、`转专业指南`、`各培养方案导读` 等。

#### 四、技术栈与特性 (The "How")

项目构建于一套精心选择的技术栈之上，旨在提供最佳的阅读、写作和协作体验：

*   **核心框架**：`MkDocs`，一个快速、简洁的静态站点生成器。
*   **主题**：`Material for MkDocs`，提供了现代化、功能丰富且高度可定制的界面。
*   **关键配置**：
    *   **语言与搜索**：全面支持中文 (`language: zh`)，并优化了中文搜索。
    *   **外观定制**：通过 `custom_dir` 和 `extra.css` 实现了高度自定义的日夜间模式和配色方案。
    *   **导航**：启用了顶部标签页导航 (`navigation.tabs`)，使得三大内容板块一目了然。
*   **Markdown 超能力 (Extensions)**：项目启用了大量的 `pymdownx` 扩展，将 Markdown 的表现力提升到了极致：
    *   **内容交互**：`admonition` (提示框), `details` (可折叠内容), `tabbed` (标签页), `tasklist` (任务列表) 等，极大地丰富了内容的呈现形式。
    *   **代码展示**：通过 `superfences`, `highlight`, `inlinehilite` 等实现了强大的代码块高亮、行号、复制和注释功能。
*   **插件系统**：
    *   `tags` 插件：构建了完善的标签系统，通过 `tags.md` 页面实现了知识点的网状连接。

#### 五、风格与写作规范 (The "Voice")

项目拥有一份明确的《内容创作与风格指南》，旨在保证所有产出内容在风格、排版和“声音”上的一致性：

*   **排版**：严格遵循**中文全角标点**和**中英文之间添加空格（“盘古之白”）** 的原则，追求专业美观。
*   **语法**：强调对 Markdown 语法的**语义化、规范化**使用。
*   **语气**：在**温度与严谨**之间寻求平衡。措辞要**包容所有贡献者**，关键页面（如首页）避免使用 Emoji 以保持专业性。
*   **特性使用**：鼓励创作者善用 `admonition` 等扩展来赋予内容恰当的“语气”和“情感”。

#### 六、协作与贡献流程 (The "Community")

项目完全基于 GitHub 进行开放式协作，拥有清晰的贡献流程：

*   **中心枢纽**：所有讨论、规划、内容修订都围绕 GitHub 仓库进行。
*   **规划与任务**：通过 `GitHub Issues` 进行内容规划和任务分配（如此次我们讨论的过程）。
*   **贡献入门**：为新人提供了 `good first issue` 标签，极大地降低了首次贡献的门槛。
*   **代码/内容提交**：遵循标准的 Fork -> Pull Request 流程。
*   **文档驱动**：项目不仅有产出文档，更有关于“如何产出”的元文档（如《风格指南》和我们这次讨论的总结），确保了协作的高效与高质量。
